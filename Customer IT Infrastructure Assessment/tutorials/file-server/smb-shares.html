SMB Shares

Tutorial – File Server Health Check

Objective

This check verifies that SMB shares are present and accessible on the File Server.

Incorrectly configured shares can impact:

  • User access
  • Applications
  • Shared data availability
  • Business operations

Prerequisites

  • Access to the File Server
  • User account with read or admin permissions
  • PowerShell access

Verify SMB Shares

Open PowerShell on the File Server and run:

PowerShell 
Get-SmbShare

Verify that:

  • Expected shares are listed
  • Share paths are valid
  • No obsolete or suspicious shares are present

Verify Share Accessibility

Open File Explorer and access the File Server shares:

Explorer 
\\ROE-vFS01

Verify that:

  • Shares are visible
  • Shares open without errors
  • Shared folders are accessible
  • ✅ Shares accessible = OK
  • ⚠️ Missing or unused shares detected = WARNING
  • ❌ Shares inaccessible or unavailable = NOK

Result Interpretation

  • ✅ OK – SMB shares correctly configured and accessible.
  • ⚠️ WARNING – Missing or suspicious shares detected.
  • ❌ NOK – SMB shares inaccessible or misconfigured.
