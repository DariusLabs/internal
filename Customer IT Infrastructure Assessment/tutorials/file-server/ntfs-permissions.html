NTFS Permissions

Tutorial – File Server Health Check

Objective

This check verifies that NTFS permissions are correctly configured on shared folders.

Incorrect permissions can lead to:

  • Unauthorized access
  • Data modification or deletion
  • Privilege escalation
  • Security risks

Prerequisites

  • Access to the File Server
  • User account with read or admin permissions
  • File Explorer access

Verify NTFS Permissions

On the File Server:

  • Right-click the shared folder
  • Select Properties
  • Open the Security tab

Verify that:

  • Permissions are assigned to groups instead of individual users
  • No unnecessary Full Control permissions exist
  • No suspicious or unknown accounts are present
  • ✅ Permissions correctly configured = OK
  • ⚠️ Excessive permissions detected = WARNING
  • ❌ Unauthorized or insecure permissions detected = NOK

Result Interpretation

  • ✅ OK – NTFS permissions correctly configured.
  • ⚠️ WARNING – Excessive or inconsistent permissions detected.
  • ❌ NOK – Unauthorized or insecure NTFS permissions present.
