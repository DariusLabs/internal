vSphere HA / Cluster Health

Tutorial – VMware Cluster Checklist Verification

Objective

This check verifies the vSphere environment health with regard to High Availability (HA).

The purpose is to confirm whether a vSphere cluster is present and whether vSphere HA is enabled, disabled, or not applicable. This is a verification-only control.

No configuration or remediation is required. The checklist item may be validated even if no cluster is configured or if vSphere HA is disabled, as long as this state has been reviewed.

Prerequisites

  • Access to vSphere Client
  • Read or monitoring permissions
  • Access to ESXi hosts and virtual machines

Open vSphere Client

Log in to the vSphere Web Client using a web browser. Access the vCenter instance via its IP address or FQDN.

Check Cluster Health and HA Status

Navigate to the vSphere inventory and review the High Availability (HA) and environment health status.

  • Inventory → Hosts and Clusters
  • If a cluster is present, select the cluster and open the Summary or Monitor tab
  • Review the Cluster Health indicators and the vSphere HA status

If no cluster is present and hosts are running in standalone mode, vSphere HA is not applicable. This state must be verified and documented.

Confirm that the health information is visible and note whether vSphere HA is enabled, disabled, or not applicable.

Result Interpretation

  • ✅ OK – Environment verified. vSphere HA enabled, disabled, or not applicable (no cluster).
  • ℹ️ Information – vSphere HA disabled by design. No action required.
  • ⚠️ Warning – Cluster health shows degraded or warning states.
  • ❌ NOK – Cluster not accessible or health status cannot be reviewed.

Note: This checklist item validates that the environment has been reviewed. It does not require a cluster or vSphere HA to be enabled.
