Last Backup Timestamp

Tutorial – Backup & Disaster Recovery

Objective

This check verifies that backup jobs are running recently and according to schedule.

Outdated backup timestamps may indicate:

  • Failed backup jobs
  • Disabled schedules
  • Backup infrastructure issues
  • Missing restore points

Prerequisites

  • Access to the backup console
  • User account with read or admin permissions

Verify Backup Timestamp

Open the backup console and navigate to:

Home → Jobs → Backup

Verify that:

  • Last Run is recent
  • Last Result is successful
  • Next Run is scheduled
  • Daily backups should normally run within the last 24 hours
  • Weekly backups should normally run within the last 7 days

Result Interpretation

  • ✅ OK – Backup jobs ran recently and successfully.
  • ⚠️ WARNING – Backup timestamps are older than expected.
  • ❌ NOK – No recent successful backups detected.
