Backup Storage Capacity

Tutorial – Backup & Disaster Recovery

Objective

This check verifies that backup repositories have enough free storage space.

Insufficient backup storage can cause:

  • Backup failures
  • Missing restore points
  • Retention issues
  • Interrupted backup operations

Prerequisites

  • Access to the backup console
  • User account with read or admin permissions

Verify Repository Capacity

Open the backup console and navigate to:

Backup Infrastructure → Backup Repositories

Verify that:

  • Repositories are online
  • Sufficient free space is available
  • No repository is close to full capacity
  • Repositories should normally keep at least 20% free space

Result Interpretation

  • ✅ OK – Backup repositories have sufficient free space.
  • ⚠️ WARNING – Repository free space is running low.
  • ❌ NOK – Repository is full or unavailable.
