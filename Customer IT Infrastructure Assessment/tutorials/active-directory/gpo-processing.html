GPO Processing

Tutorial – Active Directory Health Check

Objective

This check verifies that Group Policies (GPOs) are correctly applied on domain-joined machines.

Prerequisites

  • Access to a domain-joined machine
  • User account affected by Group Policies

Check Applied GPOs

Open Command Prompt and run:

CMD 
gpresult /r

Verify that:

  • Expected GPOs are listed
  • No denied or failed GPOs appear
  • User and Computer settings are displayed correctly

Check Group Policy Errors

Open:

  • Event Viewer
  • Applications and Services Logs
  • Microsoft → Windows → GroupPolicy → Operational

Verify that:

  • No Group Policy processing errors are present
  • No timeout or access denied events appear

Result Interpretation

  • ✅ OK – GPOs are applied correctly with no errors.
  • ⚠️ WARNING – Minor Group Policy issues or delays detected.
  • ❌ NOK – Group Policy processing failures detected.
