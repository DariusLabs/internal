AD Backup Verification

Tutorial – Active Directory Health Check

Objective

This check verifies that at least one Domain Controller is properly backed up using the organization's backup solution.

Active Directory backups are essential to recover from corruption, ransomware, accidental deletion, or major infrastructure incidents.

Prerequisites

  • Access to the backup solution
  • Read access to backup jobs or reports
  • At least one Domain Controller present in the environment

Verify Domain Controller Backups

Open the organization's backup solution (Veeam, Acronis, Nakivo, Altaro, etc.) and verify that at least one Domain Controller is included in backups.

Confirm that:

  • A recent backup of the Domain Controller exists
  • The latest backup job completed successfully
  • Backups are performed regularly
  • ✅ Recent successful backup = OK
  • ⚠️ Old or irregular backups = WARNING
  • ❌ No Domain Controller backup = NOK

Result Interpretation

  • ✅ OK – Domain Controller backups are recent and successful.
  • ⚠️ WARNING – Backups exist but are outdated or inconsistent.
  • ❌ NOK – No valid backup found for Domain Controllers.
