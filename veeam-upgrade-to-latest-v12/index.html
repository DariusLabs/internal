Veeam Backup & Replication

Upgrade Veeam Backup & Replication 12.x to Latest Version 12

🎯 Objective

Upgrade an existing Veeam Backup & Replication 12.x environment to the latest available Version 12 release before upgrading to Version 13.

This procedure preserves all existing backup jobs, repositories, proxies, credentials and configuration.

📋 Environment

Component Value
Operating System Windows Server
Source Version Any Version 12 Build
Target Version Latest Available Version 12 Release
Database PostgreSQL or Microsoft SQL Server

📌 Verify Current Version

Open:

  • Menu → Help → About

Verify the currently installed Veeam Backup & Replication build number.

Record the version before starting the upgrade.

This procedure applies to environments running an older Version 12 build that must be upgraded to the latest available Version 12 release before proceeding with additional upgrades or migrations.

⚠️ Prerequisites

  • Verify all backup jobs completed successfully.
  • Verify no backup, backup copy, tape or configuration backup jobs are running.
  • Verify Configuration Backup is enabled.
  • Verify Configuration Backup files exist on the repository.
  • Verify the encryption password used by Configuration Backup is present in Password Manager.
  • Verify all backup repositories are online and accessible.
  • Verify all backup proxies are online.
  • Verify the license is valid.
  • Verify sufficient free disk space exists on the system drive.
  • Create a VMware snapshot.
  • Download the latest available Version 12 ISO.

📦 Verify Configuration Backup

  • Open Menu → Configuration Backup.
  • Verify Configuration Backup is enabled.
  • Verify the last backup completed successfully.
  • Verify the repository is accessible.

Browse the repository and confirm that .BCO files are present.

🔐 Verify Encryption Keys

  • Open Menu → Manage Passwords.
  • Verify the encryption password used by Configuration Backup exists.
  • Verify the encryption password is currently in use.

🔑 Verify License

  • Verify License Status = Valid.
  • Verify Support Contract is active.
  • Verify no license warnings are present.

💾 Verify Free Disk Space

Verify sufficient free space exists on the system drive.

A minimum of 40 GB of available free space is recommended before starting the upgrade.

Expand the system volume if necessary.

📸 Create VMware Snapshot

  1. Verify no Veeam jobs are running.
  2. Create a VMware snapshot.
  3. Verify the snapshot completed successfully.

💿 Download Latest Version 12 ISO

  1. Log in to the Veeam Customer Portal.
  2. Open Products.
  3. Select Veeam Backup & Replication.
  4. Download the latest available Version 12 ISO.

Do not install Version 13 as part of this procedure.

Download the latest available Version 12 ISO from the Veeam Customer Portal.

If the required build is no longer easily accessible from the Veeam Customer Portal, previous Veeam Backup & Replication releases can be referenced from:

Veeam Backup & Replication Build History

Always download installation media from official Veeam sources whenever possible.

🚀 Start Upgrade

  1. Mount the ISO.
  2. Run Setup.exe.
  1. Select Upgrade Veeam Backup & Replication.
  1. Accept the License Agreement.
  1. Verify that the existing license is detected correctly.
  1. Continue through the wizard.

🔄 Reboot Server

During prerequisite installation, Veeam may require a server reboot before the upgrade can continue.

  1. Click Reboot.
  2. Wait for the server to restart completely.
  3. Reconnect to the server.
  4. Mount the Veeam installation ISO again if required.
  5. Open the mounted ISO.
  6. Run Setup.exe.
  7. Click Upgrade Veeam Backup & Replication.
  8. Go through the upgrade wizard again.
  9. Continue the installation until completion.

⚙️ Upgrade Components

Review the detected Veeam components that will be upgraded.

Verify all required components are selected and continue.

⏳ Perform Upgrade

Click Upgrade and wait for the installation to complete.

The upgrade process may take several minutes depending on the environment.

Verify the installer reports a successful upgrade.

🔄 Final Reboot

Verify that the upgrade completed successfully.

Reboot the server if requested.

🔐 First Login After Upgrade

  • Launch the Veeam Console.
  • Sign in using the current user.
  • Complete MFA authentication if required.

Microsoft Entra ID Notification

After the first launch of Veeam Backup & Replication 12.3, an informational message about Microsoft Entra ID backup support may appear.

This is expected behavior and does not require any action.

Click OK to continue.

Update Local Components

After the Veeam Backup & Replication server upgrade is completed and the console opens successfully, Veeam may detect local components that still require an update.

Select the server and click Apply.

Wait for the update process to complete.

Verify that all components have been upgraded successfully and click Finish.

🧪 Backup Validation Test

  1. Right-click an existing backup job.
  2. Select Start.
  3. Do not use Active Full.

Verify the backup completes successfully.

Run a manual Configuration Backup.

Perform a repository rescan and verify no errors are reported.

🔄 Rollback Procedure

  1. Shut down the VM.
  2. Open VMware Snapshot Manager.
  3. Select the pre-upgrade snapshot.
  4. Restore the snapshot.
  5. Power on the VM.
  6. Verify Veeam functionality.

🧹 Snapshot Cleanup

Keep the VMware snapshot until at least one backup job has completed successfully.

Delete the snapshot after validation is completed.

✅ Expected Result

  • Veeam upgraded successfully to the latest Version 12 release.
  • All backup jobs preserved.
  • All repositories accessible.
  • All proxies operational.
  • Configuration Backup preserved.
  • Test backup completed successfully.

Erreur observée

FATAL: authentification SSPI échouée pour l'utilisateur "postgres"

ou

authentication SSPI failed for user "postgres"

Cause

Le compte Windows utilisé pour lancer l'upgrade n'est pas autorisé dans PostgreSQL à se connecter avec le compte PostgreSQL postgres.

Vérification

Ouvrir le fichier :

notepad "C:\Program Files\PostgreSQL\15\data\pg_ident.conf"

Vérifier qu'une ligne existe pour le compte utilisé pendant l'upgrade.

Exemple de configuration valide :

# MAPNAME    SYSTEM-USERNAME         PG-USERNAME

veeam  Administrator@TEN-VBU01       postgres
veeam  "SYSTEM@NT AUTHORITY"         postgres
veeam  vbr.admin@TEN-VBU01           postgres

Résolution

Ajouter le compte Windows utilisé pour exécuter l'upgrade : 

veeam  NOM_UTILISATEUR@SERVEUR  postgres

Redémarrer ensuite le service PostgreSQL : 

Restart-Service postgresql-x64-15

Une fois le service redémarré, relancer l'upgrade Veeam.

Quand appliquer cette correction ?

Uniquement si l'upgrade échoue avec une erreur contenant : 

authentication SSPI failed for user "postgres"
