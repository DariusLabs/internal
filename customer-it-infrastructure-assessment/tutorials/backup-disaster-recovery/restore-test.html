Restore Test

Tutorial – Backup & Disaster Recovery

Objective

This check verifies that backup restore operations are functional.

A successful backup does not guarantee that data can actually be restored.

Restore issues can impact:

  • Disaster recovery
  • Business continuity
  • Data availability
  • Recovery operations

Prerequisites

  • Access to the backup console
  • User account with restore permissions
  • Available restore point

Start a Restore Test

Open the backup console and navigate to:

Home → Backups → Disk

Select a backup and start a restore operation.

Verify Restore Access

Verify that:

  • Restore points are accessible
  • The restore wizard opens correctly
  • No restore errors are displayed

Result Interpretation

  • ✅ OK – Restore operation can be started successfully.
  • ⚠️ WARNING – Restore operation partially works or shows warnings.
  • ❌ NOK – Restore operation fails or restore points are inaccessible.
