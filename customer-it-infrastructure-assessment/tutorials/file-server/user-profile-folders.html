User Profiles Folder

Tutorial – File Server Health Check

Objective

This check verifies that the User Profiles folder is present, accessible, and properly secured.

Incorrect profile configuration can cause:

  • User login issues
  • Profile corruption
  • Unauthorized access to user data
  • Storage problems

Prerequisites

  • Access to the File Server
  • User account with read or admin permissions
  • File Explorer access

Verify User Profiles Folder

Open File Explorer and navigate to the profiles folder.

Example:

Explorer 
\\FileServerName\Commun\DocParta

Verify that:

  • One folder exists per user
  • No suspicious or orphaned folders are present
  • Folder names match user accounts
  • Users cannot access other users’ folders
  • ✅ Profiles accessible and secured = OK
  • ⚠️ Minor permission or cleanup issues detected = WARNING
  • ❌ Insecure or misconfigured access detected = NOK

Result Interpretation

  • ✅ OK – User profiles folder correctly configured and secured.
  • ⚠️ WARNING – Minor permission or cleanup issues detected.
  • ❌ NOK – Insecure or misconfigured user profile access.
