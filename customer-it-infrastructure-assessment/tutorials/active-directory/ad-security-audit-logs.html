AD Security Audit Logs

Tutorial – Active Directory Health Check

Objective

This check verifies that Active Directory security auditing is enabled and that security events are properly logged on Domain Controllers.

Prerequisites

  • Domain account with read or admin permissions
  • Access to at least one Domain Controller
  • PowerShell or Event Viewer access

Verify Audit Policy

Open PowerShell on the Domain Controller and run: 

auditpol /get /category:*

Verify that the following categories are configured with Success or Success and Failure.

Category Expected Result
Logon Success and Failure
Account Lockout Success
Special Logon Success
User Account Management Success
Security Group Management Success
Directory Service Access Success
Kerberos Authentication Service Success
Credential Validation Success
  • ✅ Expected values present = auditing configured correctly
  • ⚠️ Missing categories = partial auditing
  • ❌ No Auditing on critical categories = auditing issue

Verify Security Log Activity

Open Event Viewer on the Domain Controller.

  • Windows Logs
  • Security

Verify that the Security log is actively recording events.

Use Filter Current Log and search for: 

4625

Event ID 4625 corresponds to failed logon attempts and is commonly present in Active Directory environments.

  • ✅ Recent events displayed = Security logging operational
  • ⚠️ Very few events = possible low audit activity
  • ❌ No events found = possible logging or auditing issue

Result Interpretation

  • ✅ OK – Security auditing enabled and events logged correctly.
  • ⚠️ WARNING – Partial auditing or insufficient event coverage.
  • ❌ NOK – Auditing disabled or no security events recorded.
