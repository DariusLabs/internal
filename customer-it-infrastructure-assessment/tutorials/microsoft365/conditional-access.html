Conditional Access

Tutorial – Microsoft 365 Security

Objective

This check verifies that Conditional Access policies are configured and enabled.

Missing Conditional Access policies can expose the tenant to:

  • Unauthorized access
  • Password attacks
  • Unsecured sign-ins
  • Identity compromise

Prerequisites

  • Access to the Microsoft Entra Admin Center
  • User account with Security Reader or admin permissions

Verify Conditional Access Policies

Open the Microsoft Entra Admin Center and navigate to:

Microsoft Entra → ID Protection → Risk-based Conditional Access → Policies

Direct Link 
https://entra.microsoft.com/#view/Microsoft_AAD_ConditionalAccess/ConditionalAccessBlade/~/Policies

Verify that:

  • Conditional Access policies are present
  • At least one policy is enabled
  • MFA protection policies exist

Important recommended policies:

  • Block legacy authentication
  • Multifactor authentication for admins
  • Multifactor authentication for all users

Result Interpretation

  • ✅ OK – Conditional Access policies are configured and critical MFA protections are enabled.
  • ⚠️ WARNING – Conditional Access exists but important protections are missing or disabled.
  • ❌ NOK – No Conditional Access policies configured or no MFA protection enabled.

Conditional Access is considered a critical Microsoft 365 security protection.
