SSL Certificate Expiration

Tutorial – Network / Security

Objective

This check verifies that the SSL/TLS certificate used by a website or service is valid, trusted and not expired.

Prerequisites

  • Access to a Windows server or workstation
  • Internet access
  • PowerShell access
  • HTTPS URL to test

Verify SSL Certificate Accessibility

Open PowerShell and run:

PowerShell 
Invoke-WebRequest https://your-domain.com

Verify that:

  • The HTTPS website is accessible
  • StatusCode returns 200
  • StatusDescription returns OK
  • No SSL or certificate warning appears
  • The certificate is trusted and valid

Expected Result

A valid SSL/TLS certificate should return:

Result 
StatusCode        : 200
StatusDescription : OK

This confirms that:

  • The website is reachable over HTTPS
  • The SSL certificate is valid
  • The certificate is trusted by Windows
  • No SSL expiration issue is detected

Common Targets

  • VPN portal
  • OWA / Exchange
  • RDS Gateway
  • Client web portal
  • Public website

Result Interpretation

  • ✅ OK – SSL certificate is valid and trusted.
  • ⚠️ WARNING – Certificate expiration is approaching or minor SSL issues detected.
  • ❌ NOK – Certificate expired, invalid, or HTTPS access failed.
